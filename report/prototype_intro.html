A Note on This Prototype

YuRong C.

This prototype was built in code, not Figma, using Claude Code. Every interaction was implemented and iterated by hand: the typing animations, the slow pulse on the plan card, the way the split view slides in only after you approve the plan, the amber highlight on a modified step.

It took longer than a static mockup. I did it because some things only become real when you can actually feel the timing. Having something live also made early testing more honest: people around me could actually use it, not just look at it.

The prototype covers two core flows: plan confirmation and auth handoff. Building them in code surfaces ideas that wireframes never do. Some of those ideas made it in. Others did not, but they are noted and form a foundation worth building on. You can see how some of the UI components evolved across iterations in the Design Evolution page.

One thing this prototype does not fully address is the conversation layer itself: giving the agent's messages more hierarchy, and creating clearer visual distinction between information and actions. This is a much deeper problem than it appears, and one I genuinely look forward to exploring further.

What you can feel here is the part that matters most: what it is like when an agent asks for your input before it acts, and what it is like when it pauses and hands control back to you cleanly.

Flow 01
Execution Plan Confirmation

Before the agent runs, it presents a step-by-step execution plan for your review. Approve it as-is, or modify individual steps before anything begins. This is the primary human-in-the-loop checkpoint.

Start here. Submit the pre-filled task in the input bar to begin.
Flow 02
Google Auth Handoff

Mid-task, the agent encounters a step requiring Google authentication. Rather than failing silently, it pauses, explains why it needs your help, and hands control back cleanly.

Skip ahead. Use the Demo button in the input bar to jump directly to the auth handoff state.
