04 / 20
Human-AI Collaboration

The ideal: a seamless loop between human and agent.

Each phase hands off cleanly — the agent works autonomously, surfaces decisions at the right moment, and the human stays in control without micromanaging.

01
Research
Agent browses, gathers, synthesises autonomously
→
Human in loop
02
Judgement
Human reviews plan, approves or redirects before execution
→
Human in loop
03
Auth handoff
Human grants or denies access; agent proceeds with clarity
→
04
Delivery
Output handed off cleanly; human can act immediately

In practice, each handoff point is a potential breakpoint — and most products leave users stranded there.

05 / 20
Design Challenge

AI will fail.
The goal isn't zero failures —
it's designed recovery.

Bad experience The user has no idea what went wrong or what to do next. Trust breaks.
Good experience The failure is visible, named, and recoverable. The user stays in control.

"Errors and failures are an inevitable part of any product experience."

Google · People + AI Guidebook, Errors & Graceful Failure
06 / 20
The finding

Users lose control at every critical moment.

The black-box nature of the workflow strips the user of control. These gaps generalize to any agent platform.

Before execution
  • No plan preview before the agent starts
  • VM panel front-loaded before the user types anything
During execution
  • Running, stuck and failed all look identical
  • Auth prompt loops with no recovery path
  • Permission UI shows raw JS to a non-coder
After completion
  • Step-limit stop leaves no partial output, no summary
  • Final deliverable buried in a wall of text
07 / 20
This is not a Simular bug

Human-in-the-loop is a blind spot across the category.

When agents need human input or want to show progress, most lack the mechanism to do it well.

Pattern A
"Are you still working?" — user loses track of progress

Execution stream updates constantly but has no hierarchy. Running, stuck, and failed all look the same.

Pattern B
The agent asks to run code — but the user can't tell if it's safe

Raw code shown to a non-technical user. No way to judge if it's safe.

Pattern C
Hard stop with no handoff

Task ends abruptly. No summary, no partial output, no path forward.

08 / 20
The worst moment

The AI agent stuck in a dead loop.

An agent should know when to check human intention when it goes wrong.

Design Strategy

The Human-in-the-Loop Spectrum

HITL
Human-in-the-Loop

Human reviews and approves each critical decision.

Current necessity.
HOTL
Human-on-the-Loop

Human monitors and can intervene when needed.

Near-term target.
HOOL
Human-out-of-the-Loop

Fully autonomous. Human sets goal, receives result.

Long-term vision.
Legibility Make the agent's plan readable before execution. Users can't trust what they can't see.
Boundary Awareness Agent recognises its own limits and hands off cleanly. No silent failure loops.
Controllability Users steer without disrupting. Deliberate intervention, never accidental takeover.
Progressive Disclosure Complexity appears only when needed. The interface matches the user's growing mental model.
13 / 20
01 Plan Pre-Confirmation

Give users expectation with plan before execution.

Audit gap · No plan preview

A plan card surfaces before execution — users see steps, auth moments, and decide before anything runs.

Before

Undifferentiated stream. Plan, decisions, and output live in the same flow — the user can't find anything at a glance.

After

Reviewable card. Numbered steps with estimated handoff moments flagged upfront — users know what the agent will need from them before anything runs.

14 / 20
02 Auth Handoff & Graceful Fallback

Graceful fallback when plan changes.

Audit gap · Silent retry loop

Auth becomes a deliberate pause — plain-language intent, clear recovery path if the user says no.

Before

The user said no. The agent ignored it — same prompt, again and again. No exit.

After

Cancelling is a valid choice. Agent acknowledges, holds state, and offers options — retry, modify, or skip.

15 / 20
03 Workspace Transparency & Safe Control

Structured workspace for users to watch or control.

Audit gap · Accidental VM takeover

Tabs show the agent's live context; a takeover dialog makes control transfer explicit and intentional.

Before

Raw code, no context. The user can't evaluate what they're approving or why.

After

Plain-language intent. What the agent wants, why, and exactly what the user is agreeing to.

16 / 20
04 Progressive Reveal

Progressively reveal info when it's needed.

Audit gap · Front-loaded VM on homepage

The interface starts with a single focused input — the workspace appears only after the plan is approved.

Before

Front-loaded complexity. Task list and VM panel loaded immediately — overwhelming before a single word is typed.

After

One thing at a time. Clean input first; workspace and split view reveal only when needed.

17 / 20
05 Post-Completion: Structured Delivery

Every task ends with a clear handoff.

Audit gap · Output buried in VM

Task completion surfaces a structured card with output, export options, and next actions — not just "done."

Before

Text "done", nothing else. User must hunt for output inside the VM — no summary, no next step.

After

File card + next actions. What was done, what was saved, and how to use it — all visible immediately.

18 / 20
06 Task Progress Sidebar

Transparent steps show where you are.

Audit gap · All states look identical

A collapsible sidebar shows steps with status, a "needs your attention" signal, and expandable detail — so running, stuck, and failed no longer look the same.

Before

Undifferentiated stream. Every step looks the same — users can't tell what's done, what's running, or what needs them.

After

Structured progress sidebar. Steps with status, attention signals, and collapsible detail — glanceable at any moment.
