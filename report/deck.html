01 / 18 · Human + Agent
The thesis

AI agents are shipping similar capabilities.
UX is what differentiates and retains — and human-in-the-loop is where it's won or lost.

When every platform can run an autonomous agent, the product that keeps users isn't the one with more capability. It's the one that handles the messy moments — waiting, permission, failure, handoff — without losing the human.

Human-AI Collaboration

The ideal: a seamless loop between human and agent.

Each phase hands off cleanly — the agent works autonomously, surfaces decisions at the right moment, and the human stays in control without micromanaging.

03 / 18
Design Challenge

AI will fail.
The goal isn't zero failures —
it's designed recovery.

"Errors and failures are an inevitable part of any product experience."

Google · People + AI Guidebook, Errors & Graceful Failure

Failure is not the edge case — it's part of every run. The real design question is: what does the user experience in that moment, and can they find their way forward?

Without design User hits a wall — no context, no options, no way back. They leave.
With design User sees what happened, understands why, and has a clear next step. They stay.

This is not a Simular bug

Human-in-the-loop is a blind spot across the category.

I ran the same hard task across several autonomous agents. Each one stumbled at the human-in-the-loop moments — in its own way. The failure modes differ; the weak spot is shared.

Pattern A
No visibility into progress

"Is it still working, or stuck?" — users can't tell.

Pattern B
Auth loops with no recovery

User says no — agent keeps asking. No graceful next step.

Pattern C
Hard stop — no handoff

Task ends abruptly. No summary, no partial output, no path forward.

05 / 18
The finding

Three user pain points — one per phase of the run.

The black-box nature of the workflow strips the user of control. These gaps generalize to any agent platform.

Before execution
  • No plan preview before the agent starts
  • A VM panel front-loaded before the user types anything
During execution
  • Running, stuck and failed all look identical
  • Clicking the VM transfers control by accident
  • Auth prompt loops with no recovery path
  • Permission UI shows raw JS to a non-coder
After completion
  • Step-limit stop leaves no partial output, no summary
  • Final deliverable buried in a wall of text

Design Framework

Not every moment needs the same level of human involvement.

The key is matching the interface to the stakes of the moment — from quietly keeping the user informed, to handing control back entirely.

Low stakes
Out of the loop
Inform

Keep the user oriented without demanding action. Show progress, current stage, and reasoning in the background.

Agent acts · Human watches
Medium stakes
On the loop
Confirm

Surface the plan and key decisions as reviewable cards. Easy to approve, easy to redirect before the agent moves.

Human reviews · Agent proceeds
High stakes
In the loop
Control

Permission, auth, and takeover handled deliberately — with plain-language intent and a clear path back if the user says no.

Human decides · Agent waits

10 / 18
Design Highlight
01 Plan Pre-Confirmation

See the plan before the agent moves.

Audit gap · No plan preview Principle · Progressive Disclosure

Before execution, users had no idea what the agent was about to do, which data it would touch, or when it would need their help. A plan preview creates shared expectations upfront — and flags auth handoff steps before they arrive as surprises mid-task.

Trade-off

Not every agent needs this checkpoint. But for agents still maturing in reliability, a 10-second plan review catches misalignments early and builds the trust that earns autonomy over time.

Before

Undifferentiated stream. Plan, decisions, and output live in the same flow — the user can't find anything at a glance.

After

Reviewable card. Numbered steps, handoff moments flagged upfront. Approve or redirect before anything runs.

11 / 18
Design Highlight
02 Auth Handoff & Graceful Fallback

Every "no" has a designed next step.

Audit gap · Silent retry loop Principle · Controllability

The original flow pushed OAuth through a desktop popup outside the VM. Dismissing it caused the agent to retry endlessly with no explanation and no escape. The redesign treats auth as a deliberate handoff: the agent pauses, explains what it needs, and gives the user a clear recovery path if they say no.

Trade-off

Less seamless than a fully automatic flow. But an agent that pauses cleanly and recovers gracefully is far more trustworthy than one that loops silently until it times out.

Before

The user said no. The agent ignored it — same prompt, again and again. No exit.

After

Cancelling is a valid choice. Agent acknowledges, holds state, and offers options — retry, modify, or skip.

12 / 18
Design Highlight
03 Workspace Transparency & Safe Control

Watch without accidentally taking over.

Audit gap · Accidental VM takeover Principle · Boundary Awareness

Clicking into the VM silently transferred control. Raw JS permission dialogs exposed code that non-technical users couldn't evaluate. Two decisions solve this from both angles: tabs make the agent's context visible in real time; a takeover dialog makes control transfer explicit and intentional.

Trade-off

More interface surface area. But for a product operating autonomously in a live environment, visibility without interference is the core trust mechanism.

Before

Raw code, no context. The user can't evaluate what they're approving or why.

After

Plain-language intent. What the agent wants, why, and exactly what the user is agreeing to.

13 / 18
Design Highlight
04 Progressive Reveal

Show complexity only when it's earned.

Audit gap · Front-loaded VM on homepage Principle · Legibility

The original interface opened with a split-screen and a live VM preview before the user had typed a single word. For new users, this is immediately overwhelming. The redesign starts with a single focused input — the workspace appears only after the plan is approved, matching the user's growing mental model step by step.

Trade-off

Power users may want immediate workspace access — a shortcut is available. But the default prioritises the first-time experience. Progressive disclosure doesn't hide capability; it sequences it.

Before

Front-loaded complexity. Task list and VM panel loaded immediately — overwhelming before a single word is typed.

After

One thing at a time. Clean centred input first. Workspace reveals only after the plan is approved.

14 / 18
Design Highlight
05 Post-Completion: Structured Delivery

Every task ends with a handoff, not a halt.

Audit gap · Output buried in VM Principle · Legibility

A text "done" message forced users to hunt for their output inside the VM. The redesign treats task completion as a structured delivery moment: file card, export options, and next actions all surface in one view — immediately usable without digging through the interface.

Trade-off

Adds a step after the main task. But showing what was actually produced — and giving the user immediate ways to use it — is the difference between completing a task and delivering value.

Before

Text "done", nothing else. User must hunt for output inside the VM — no summary, no next step.

After

File card + next actions. What was done, what was saved, and how to use it — all visible immediately.

15 / 18
Design Highlight
06 Task Progress Sidebar

Make "running" look different from "stuck."

Audit gap · All states look identical Principle · Legibility · Controllability

The original execution stream updated constantly but had no visual hierarchy. Agent reasoning, process logs, and required actions all appeared as undifferentiated text. Running, stuck, and failed states looked identical. The sidebar introduces a collapsible progress view — steps with status, a clear "needs your attention" signal, and the ability to expand or collapse detail as needed.

Trade-off

More UI surface. But hiding progress entirely leaves users with no way to know if the agent is working, waiting, or broken — and that uncertainty erodes trust faster than any extra chrome.

Before

Undifferentiated stream. Every step looks the same — users can't tell what's done, what's running, or what needs them.

After

Structured progress sidebar. Steps with status, attention signals, and collapsible detail — glanceable at any moment.
